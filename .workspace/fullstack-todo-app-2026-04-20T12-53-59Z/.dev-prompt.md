### Description
# Todo Application

Build a single-page todo application using Vue 3 with TypeScript.

## Requirements

Users should be able to:

1. **Add a todo** — Type text into an input field and submit (button click or
   Enter key) to add a new todo item to the list.

2. **View todos** — See all todo items in an ordered list. Each item shows its
   text content and a way to mark it complete.

3. **Complete a todo** — Click a checkbox to toggle a todo between active and
   completed states. Completed todos should be visually distinct (e.g.,
   strikethrough text, different styling).

4. **Delete a todo** — Remove a specific todo from the list entirely.

5. **Clear completed** — Remove all completed todos from the list with a single
   action.

6. **Item count** — Display the number of remaining (active) todos.

## Behavior Details

- The input field should clear after successfully adding a todo.
- Empty submissions (blank or whitespace-only) should be ignored.
- New todos are added to the end of the list.
- New todos start in the active (not completed) state.
- The item count updates immediately when todos are added, completed, or deleted.
- &quot;Clear completed&quot; button should only be visible when completed todos exist.

## Technical Constraints

- Use Vue 3 Composition API with &#x60;&lt;script setup&gt;&#x60; syntax
- Use TypeScript
- Use semantic HTML (header, main, footer, nav, form, ul&#x2F;li, etc.)
- All interactive&#x2F;testable elements must have &#x60;data-testid&#x60; attributes
  as specified in the contract
- No external state management library — use &#x60;ref&#x60;&#x2F;&#x60;reactive&#x60; from Vue
- No routing required — single page

## State Management

All state lives in &#x60;App.vue&#x60; using Vue 3 &#x60;ref&#x60;:

&#x60;&#x60;&#x60;typescript
const todos &#x3D; ref&lt;Todo[]&gt;([]);

Pass state down via props. Emit events up for mutations:

- &#x60;TodoForm&#x60; emits &#x60;add(text: string)&#x60;
- &#x60;TodoItem&#x60; emits &#x60;toggle(id: number)&#x60; and &#x60;delete(id: number)&#x60;

## Key Patterns

- **Computed properties** for derived state:
  - &#x60;activeTodos&#x60; — filter where &#x60;completed &#x3D;&#x3D;&#x3D; false&#x60;
  - &#x60;completedTodos&#x60; — filter where &#x60;completed &#x3D;&#x3D;&#x3D; true&#x60;
  - &#x60;activeCount&#x60; — length of &#x60;activeTodos&#x60;
  - &#x60;hasCompleted&#x60; — boolean, controls &quot;clear completed&quot; visibility

- **Event handling** in App.vue:
  - &#x60;addTodo(text)&#x60; — push new Todo, increment nextId
  - &#x60;toggleTodo(id)&#x60; — flip completed boolean
  - &#x60;deleteTodo(id)&#x60; — filter out by id
  - &#x60;clearCompleted()&#x60; — filter to only active todos
